## Что такое CI

CI (Continuous Integration) — это автоматический запуск тестов при каждом push или pull request.

Цель:
- автоматически проверять корректность кода
- не допускать попадания “сломанных” изменений в main
- снижать риск ошибок и уязвимостей

CI — базовый элемент DevOps и DevSecOps практик.

---

## Что реализовано

В проект добавлен GitHub Actions workflow:

Путь:
.github/workflows/tests.yml

Workflow выполняет:

1. Checkout репозитория
2. Установку Python 3.11
3. Установку pytest
4. Запуск тестов:
   python -m pytest -q

Workflow запускается при:
- push
- pull_request

## Почему это важно для ИБ-проектов

CI позволяет:

- предотвращать попадание нестабильного кода в продакшен
- контролировать целостность проекта
- автоматически проверять изменения
- формировать безопасный процесс разработки

В контексте информационной безопасности это снижает риск внедрения уязвимого кода.
